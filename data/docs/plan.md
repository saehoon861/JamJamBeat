# 동영상 프레임 단위 수동 라벨링 툴 개발 기획 및 세부 구현 계획서

## 1. 개요 및 목표
* **목표**: `data/raw_data/` 폴더 내 위치한 핸드 제스처 동영상(720p, 30fps 고정)을 프레임 단위로 수동 라벨링하기 위한 파이썬 기반 도구 개발
* **입력 데이터**: `{클래스ID}_{템포}_{왼손오른손}_{참여자}.mp4` 형식의 영상 구동
* **출력 데이터**: 라벨링된 결과를 프레임 단위로 꼼꼼히 기록한 CSV 파일 포맷 생성

## 2. 라벨 매핑 및 파이프라인 구조 개념
현재 정의된 제스처 라벨 클래스는 다음과 같습니다.
* `0`: Neutral (평상시 손, 무의미한 움직임)
* `1`: Fist (주먹. 주먹을 꽉 쥐는 동작)
* `2`: Open Palm (손바닥. 손바닥을 활짝 펴는 동작)
* `3`: V (브이. 검지와 중지만 펴는 브이 자세)
* `4`: Pinky (약속. 새끼손가락만 펴는 자세)
* `5`: Animal (여우 손. 중지+약지를 엄지에 맞닿게, 검지+새끼 쭉 펼침)
* `6`: K-Heart (하트. 한 손으로 엄지+검지를 교차해 만드는 하트)

> **[유지보수 및 확장성 설계]**
> 위 매핑 규칙은 소스 코드 내부에 하드코딩하지 않습니다. `config.yaml`과 같은 별도의 설정 파일을 두고, 파일명에서 추출한 클래스 ID와 설정 파일의 명칭을 동적으로 매치시킵니다. 이를 통해 추후 동작 종류가 확장되거나 변경되어도 설정 파일 수정만으로 대응하도록 구성합니다. 하단 타임라인 바에 사용될 라벨별 색상(`label_colors` 속성, 예: 0=회색, 1=빨강...)도 `config.yaml`에 함께 정의합니다.

## 3. 결과물 출력 포맷 
영상이 **30fps** 고정임을 활용하여 타임스탬프를 수학적으로 계산하고 기록합니다. 
(1프레임당 소요 시간: 약 `33.033ms`)
프레임 인덱스는 0부터 시작하며, 0번 프레임은 `00:00:000`의 타임스탬프를 가집니다.

| frame_idx | timestamp | gesture | 비고 |
| :-- | :-- | :-- | :-- |
| 0 | 00:00:000 | 0 | 라벨링 초기 기본값 (가장 기본은 0번 Neutral 입니다) |
| 1 | 00:00:033 | 0 | |
| 2 | 00:00:066 | 0 | |
| ... | ... | ... | |
| 26 | 00:00:858 | 1 | 사용자가 라벨링한 특정 프레임 (예: 1번 Fist로 지정한 구간) |

## 4. 인터페이스(UI) 및 단축키 동작 명세
가볍고 빠른 조작을 위해 무거운 GUI 라이브러리 말고 **OpenCV 화면 출력과 이벤트 리스너(`waitKey(0)`)**만을 사용한 윈도우 인터페이스를 구성합니다.

### A. 화면 정보 표시 (OSD: On-Screen Display)
영상 화면 상단에 `cv2.putText`를 이용해 가이드를 표시합니다.
* **좌측 상단 (프레임 및 시간 정보)**
  * `Frame: {현재 프레임 확인} / {비디오의 총 프레임 수}`
  * `Time: {현재 시간} / {총 시간}` (반드시 `분:초:ms` 의 형태로 표시, 예시: `00:00:858 / 00:30:000`)
* **우측 상단 (라벨링 상태)**
  * `Current Label: {클래스ID}_{매칭된 클래스명}` (예: `Current Label: 1_Fist`)
* **화면 내 단축키 사용법 표시**:
  * 사용자가 시각적으로 조작법을 바로 알 수 있도록 제어 키 (`A: 이전 프레임`, `D: 다음 프레임`, `Q: -10 프레임`, `E: +10 프레임` 등) 가이드를 모서리에 상시 표시합니다.
* **S 마커 시각 피드백 제공**:
  * 사용자가 `S` 키를 눌러 구간 마킹을 시작한 경우, 화면 상단에 `[RANGE MODE: S marked at frame 26]` 과 같은 안내 텍스트를 눈에 띄게 추가 표시하여 직관적으로 상태를 알 수 있게 합니다.
* **동적 경고 문구(Warning Message) 표시 지원**:
  * 잘못된 조작(예: S 마커 활성 중 Enter 저장 시도) 발생 시 사용자가 인지할 수 있도록, 상태 객체나 컨트롤러에 보관된 **경고 메세지 문자열(예: `[WARNING: Cannot save. Press 'F' to close the range or 'ESC' to abort]`)** 을 일시적으로 화면 중앙이나 하단 등 눈에 띄는 붉은색 텍스트로 오버레이합니다. 이 경고 문구는 다음 키 입력이 발생하여 렌더링 사이클이 돌 때 초기화(제거)됩니다.

### B. 탐색 조작 (Navigation)
* **`D` 키 (또는 `→`)**: 다음 프레임 (+1)
* **`A` 키 (또는 `←`)**: 이전 프레임 (-1) 
* **`E` 키**: 앞으로 10 프레임 크게 건너뛰기
* **`Q` 키**: 뒤로 10 프레임 크게 건너뛰기

### C. 라벨링 제어 (Labeling)
기본적으로 시동 시 모든 프레임 라벨은 `0` 으로 초기화됩니다.
영상 파일명 파싱을 통해 자동으로 해당 영상의 "목표(Target) 제스처 번호"가 세팅됩니다.

* **`Space bar` (단일 프레임 토글)**: 
  * 누를 때마다 현재 진행 중인 프레임의 라벨 수치가 `0` ↔ `[Target 제스처 번호]` 로 전환됩니다.
* **`S` 및 `F` 키 (구간 라벨링 조작 매핑)**
  * **`S` (Start)**: 동작이 개진하는 시작 프레임을 일시정지 상태에서 지정 마킹.
  * **`F` (Finish)**: 동작이 종료되는 끝 프레임을 지정 마킹.
    * **(주의사항)**: S 키를 누르지 않은 상태에서 F 키를 단독으로 눌렀을 때는 아무 것도 덮어씌워지지 않으며, 에러 발생 없이 **조용히 무시(Pass)** 처리됩니다.
  * **(중간 덮어쓰기 로직)**: S를 누르고 사용자가 프레임을 탐색하다가 정상적으로 F를 누르면 그 사이의 프레임들은 **예외 없이 전부** Target 제스처로 덮어쓰기 처리됩니다. 중간에 스페이스바(단일 제어) 토글을 통해 변경된 프레임 이력이 있었더라도 구간 지정 명령이 강력한 우선순위를 가지며 전부 초기화/덮어쓰기 됩니다.

### D. 하단 타임라인 바 (Timeline Strip)
영상 편집 툴처럼 현재 프레임을 중심으로 전후 프레임들의 라벨링 상태를 시각적으로 확인할 수 있는 타임라인을 제공합니다.
- **영역**: 화면 하단 고정 영역 (높이 약 40px 수준)에 직사각형 형태로 렌더링합니다.
- **표시 범위**: 현재 프레임 기준 앞뒤 각 N프레임(초기 설정값 예: ±60 프레임)을 픽셀 단위로 가로 매핑합니다.
  - **경계 프레임 예외 처리**: 동영상의 시작(0번 프레임 부근) 또는 끝부분 등 가용 범위를 벗어난 영역은 타임라인 UI 상에서 **검은색(Blank)** 블록으로 그리도록 빈 공간을 명시적으로 처리합니다.
- **색상 표시**: 각 프레임 위치를 현재 라벨 상태에 맞는 고유 색상(config.yaml에 `label_colors`로 매핑해둔 색상)으로 채운 얇은 세로 직사각형 블록으로 표시합니다. (예: 0번 Neutral은 회색, 1번은 빨강, 2번은 초록 등)
- **현재 프레임 커서**: 타임라인 정중앙(현재 프레임 위치)에 굵은 흰색 세로선 커서를 그려 현 위치를 식별 가능하게 합니다.
- **S 마커 연동 표시**: S 마커가 활성화된 범위 내에 존재할 경우, 타임라인 바 위에도 S 마커가 찍힌 프레임 위치를 노란색 세로선 등으로 눈에 띄게 추가 표시하여 시각 피드백을 한층 더 강화합니다.
- **구현 및 창 병합 방식**: 순수한 원본 프레임(예: 720p 해상도 `1280x720`)과, 본래 영상 이미지와 분리된 캔버스 형태의 타임라인 바(가로 길이가 원본과 동일한 `1280x40`)를 생성한 뒤, `numpy.vstack()` 배열 결합 함수를 이용하여 위아래로 이어 붙입니다. 최종적으로 리턴 객체는 영상과 타임라인이 합쳐진 단일 `numpy` 배열이며, 이 상태로 `cv2.imshow()` 에 투입됩니다.

### E. 진행 및 저장/중단 제어 (강력 통제)
* **`Enter` 키 (구간 저장 및 다음으로 이동)**
  * 영상을 끝까지 확인하고 작업이 만료되었을 때 누릅니다. 파일명이 동일한 확장자(`.csv`)를 생성하여 저장 후 다음 영상 루프로 이동합니다.
  * **예외 처리 (S 불완전 마킹 상태 시 저장 방지)**: 만약 'S' 마커가 활성화된 상태에서(F 키로 구간을 닫지 않은 상태) Enter를 눌러 저장을 시도할 경우 파일 저장이 **Lock 거부**됩니다.
  * **경고 발생**: 이때 조용히 무시하지 않고, 컨트롤러 변수에 "경고 메세지"를 상태 주입한 뒤 화면 업데이트 과정을 1회 더 촉발시켜, 사용자가 즉시 에러 사유(`Need to close S marker`)를 화면에서 선명하게 볼 수 있도록 안내합니다.
* **`ESC` 키 (작업 즉시 중단 및 오염 방지)**
  * 사용자가 동영상을 끝까지 보지 않고 불완전한 중간 상태에서 끌 때 사용합니다.
  * 프로그램을 즉시(`sys.exit()`) 종료시키며, 현재 진행 중이던 상태 배열은 단 하나도 파일 단위로 기록/저장되지 않습니다. 동영상 하나 분량을 무조건 완수해야만 저장 가능하도록 권한을 보호합니다. (S 마커 상태 역시 무시되고 종료 처리됩니다.)

## 5. 단계별 세부 구현 내용 및 소프트웨어 아키텍처 (로드맵)
유지보수 및 확장이 쉽도록 **MVC (Model-View-Controller) 패턴**을 기본 뼈대로 차용합니다. (Controller에서 Model과 View를 명시적으로 조율)
기존 `src/` 폴더는 사용하지 않으며 모든 소스코드 개발과 모듈화 환경 구성은 전적으로 격리된 `data/` 내부의 전용 구조 하위에서만 구동되게 적용합니다.

### 5.1. 디렉토리 구조 및 파이썬 uv 독립 가상환경 적용 (격리)
파이썬 패키지 및 라이브러리 관리는 글로벌 `pip`를 사용하지 않고, 데이터 수동 라벨링만을 위한 **격리된 `uv` 가상환경**을 구축합니다.
```text
JamJamBeat/
├── data/
│   ├── plan.md                       # 기획서 포맷 문서
│   ├── raw_data/                     # 처리해야 할 .mp4 동영상 원본 보관 장소
│   ├── labeled_data/                 # 생성된 전체 CSV 결과물 파일들이 적재될 폴더
│   └── labeling_tool/                # ✨ 라벨링 툴 메인 소스코드 패키지 
│       ├── .venv/                    # [자동생성] uv 매니저를 통한 라벨링 툴 전용 가상환경
│       ├── pyproject.toml            # uv 관리를 위한 프로젝트 의존성 파일
│       ├── uv.lock                   # uv 패키지 매니저 Lock 파일
│       ├── config/
│       │   └── label_config.yaml     # 클래스 ID↔Name 매칭 정보, 타임라인용 label_colors 색상 코드 관리
│       ├── core/
│       │   ├── annotation_state.py   # [Model] 데이터 상태, Toggle 및 Range 덮어쓰기 배열 데이터 제어 책임
│       │   ├── class_pipeline.py     # [Singleton] yaml 파싱, 전역적 확장 가능 라벨 딕셔너리 및 파일명 타겟 파싱 관리
│       │   └── video_handler.py      # OpenCV VideoCapture Read/Set 프로퍼티 래핑 모듈
│       ├── ui/
│       │   └── osd_renderer.py       # [View] 프레임 이미지 위 Text Overlay(OSD) 및 하단 타임라인 바(Timeline Strip) 결합 비주얼 렌더링
│       ├── io/
│       │   ├── data_exporter.py      # Pandas 데이터프레임을 활용한 CSV 파일 Write/Dump 책임
│       │   └── file_scanner.py       # 남아있는 .mp4 스캔, 기 처리된 csv 파일 대조 필터링 및 미작업 리스트 도출
│       └── main.py                   # [Controller] OpenCV waitKey(0) 이벤트 루프 제어, 진입점(Entry Point) 및 객체 오케스트레이션
```

### 5.2. 모듈 별 세부 구현 방안 및 로직 디자인

#### 1) `labeling_tool/core/class_pipeline.py` (설정 및 제스처 라우팅 - Singleton)
* **역할**: 전역적(Global)으로 한 번만 설정(`yaml`)을 로드하여 여러 모듈에서 일관되게 Name 및 Color 정보 속성을 접근하도록 싱글톤 패턴으로 구현합니다.
* **내부 함수/로직**:
  * `class LabelConfigManager:`
    * `load_config()`: `yaml` 로드 (하드코딩 Python Dictionary 배제 원칙).
    * `get_class_name(id: int) -> str`: 클래스 ID 기반 매칭된 텍스트 반환.
    * `get_class_color(id: int) -> tuple`: 타임라인 표시 렌더링을 위한 BGR 색상 튜플 반환.
    * `parse_target_from_filename(filename: str) -> int`: `"1_slow_Left_woman1.mp4"` 등의 문자열 스템(Stem) 영역에서 첫 `_` 기준 분리를 적용하여, 맨 앞의 `1` 값을 Target으로 도출.

#### 2) `labeling_tool/core/annotation_state.py` (라벨 상태 추적 - Model)
* **역할**: 구동되는 영상 단일 건에 종속된 라벨링 정보(프레임별 최종 라벨 할당 리스트)의 단일 상태 공급 체계 구비.
* **내부 함수/로직**:
  * `class AnnotationState:`
    * 멤버로 총 프레임 길이에 상응하는 `total_frames`의 `0` 배열 선제 보유.
    * 상태 보존 속성 `start_marker` (초기값: `None`) 보유.
    * `toggle_frame(index: int)`: 해당 `index` 리스트 배열 원소를 `0` ↔ `Target ID` 로 상호 치환 적용.
    * `set_range_start(index: int)`: S를 눌렀을 때 내부 `start_marker = index` 정수값 할당.
    * `set_range_end(index: int, target_class: int)`: F를 눌렀을 때, `start_marker`가 `None`이면 조용히 넘어가고, 아니라면 `slice(start_marker, index + 1)`을 이용해 **종료 프레임인 index 자신까지 완전히 포함하게끔** 파이썬 리스트 슬라이싱 범주를 명시하여 덮어쓰기를 일괄 단행(`target_class` 값으로 전부 변경). 덮어쓰기 이후 `start_marker = None`으로 리셋.
    * `is_range_active() -> bool`: S 마커가 현재 켜져 있는지 확인하여 Enter 통과 가능 여부를 판단하는 데 사용.

#### 3) `labeling_tool/ui/osd_renderer.py` (화면 마커 및 타임라인 렌더링 - View)
* **결합 원칙**: OpenCV의 시각 처리 종속성을 완전히 분리합니다. 결과물 리턴 객체는 `1280x720` 원본 뷰어와 `1280x40` 타임라인을 `np.vstack`으로 병합한 매트릭스입니다.
* **내부 함수/로직**:
  * `class OSDRenderer:`
    * `_format_time(frame_idx: int) -> str`: 수식 `(idx / fps) * 1000` 을 활용해, 고정 포맷의 `분(00):초(00):ms(000)` 형식 형태 문자열 반환.
    * `draw_ui(...)`: 수신받은 OSD 인자(단축키 매핑, 시간/프레임 상태)와 오류가 난 Controller가 보유한 **경고 문구(Warning Message)** 가 있다면 화면 정중앙에 시인성 있게 추가 오버레이.
    * `draw_timeline(...)`: 폭 `1280x40`의 검은 numpy 캔버스 생성 후 타임라인 로직 적용. 현재 `frame_index`를 중심으로 `±60` 픽셀을 순회하되, `index < 0` 이거나 `index >= total_frames` 범위는 그리지 않고 검은 빈 공간으로 둡니다. 각 정답은 `label_colors` BGR 값을 활용해 채우며 중앙의 화이트/황색 포인터 라인을 함께 그려서 반환.

#### 4) `labeling_tool/io/file_scanner.py` 및 `io/data_exporter.py` (단말 입출력 모듈)
* **`file_scanner.py`**: `glob` 또는 `os.listdir`을 활용하여 `data/raw_data/*.mp4`의 가용 파일을 나열. `data/labeled_data/` 위치에 동일한 **파일명 형태(확장자만 `.mp4`에서 `.csv`로 변상한 stem 단위 절대 일치 비교 기준)**의 객체가 성립재하는지 1:1 대조합니다. 이 매칭 과정에서 발견된 기 처리 파일 군은 대상 큐에서 영구 필터 스킵합니다.
* **`data_exporter.py`**: 완성된 Model 주입 데이터를 `pd.DataFrame` 구조로 결합시킨 뒤, 약속된 무정형 제어 없이 지정된 Target `csv` 파일로 내보내는 Write 작업 전담.

#### 5) `labeling_tool/main.py` (시스템 오케스트레이션 - Controller & 진입점)
* **진입 명세**: 본 도구를 구동하기 위한 최종 실행 진입점(Entry Point)입니다. 터미널 인터페이스에서 `uv run data/labeling_tool/main.py` 명령어를 통해 구동합니다.
* **로직 주요 흐름**:
  1. `file_scanner`에 요청해 미작업 동영상 스캔 (분석 기준: `.mp4` ↔ `.csv` 동일 파일명 stem 매칭).
  2. `for video_path in pending_videos:` 순회 진입.
  3. `VideoHandler`와 `AnnotationState` 모델 인스턴스, `OSDRenderer` View 제어 객체 주입 생성.
  4. **최초 진입 렌더링 강제 실행**: `while True:` 블록 진입 직전, 사용자의 키보드 입력이 대기 상태인 `waitKey(0)`에 블록킹 당하기 전 화면이 먼저 한차례 표출되도록 `OSDRenderer.draw_ui()` 호출 및 `cv2.imshow()` 갱신 루틴을 명시하여 최초 시야를 확보합니다. 
  5. 무한 대기 `while True:` 블록 진입. `cv2.waitKey(0) & 0xFF`로 입력 키 입력을 Listen.
  6. **동작 판별 분기**: 
     - 루프가 돌 때마다 기존에 저장되어 있던 UI 경고 문구 변수(`warning_msg`)는 공백으로 초기화하여, 직전 턴에 띄운 에러 안내가 사라지도록 보장합니다.
     - 키 입력 판별 결과 (`A/D/Q/E`) -> 프레임 인덱스 증감 분기 트리거. 
     - 입력 연산 트리거 (`Space/S/F`) -> `AnnotationState` 객체 메쏘드를 호출.
     - Enter 제어 -> `model.is_range_active()`가 `True`일 시 저장을 막고, `warning_msg = "Need to close S marker!"` 를 할당 후 무시(`continue` 하여 렌더링 갱신). `False`일 시 최종 저장(`data_exporter` 호출) 후 `break`.
     - ESC 제어 -> 즉시 파기(`sys.exit`).
  7. 키 입력을 통해 상태값이 업데이트된 뒤, `draw_ui` 호출하여 합성된 프레임(원본+`np.vstack` 타임라인 결합 뷰)을 재호출하여 `cv2.imshow()`로 갱신 완료.
