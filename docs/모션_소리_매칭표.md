# 모션과 소리 매칭표

## 개요

이 문서는 JamJamBeat 프로젝트에서 손동작(제스처)이 어떻게 소리로 변환되는지 전체 매핑 구조를 설명합니다.

---

## 1. AI 모델이 인식하는 손동작 (7가지)

모델은 시퀀스 기반으로 8프레임의 손 랜드마크 데이터를 분석하여 다음 7가지 클래스를 분류합니다.

| 클래스 ID | 모델 라벨 | 코드 라벨 | 이모지 | 설명 |
|-----------|-----------|-----------|--------|------|
| 0 | `neutral` | `None` | - | 중립 상태 (손동작 인식 안됨) |
| 1 | `fist` | `Fist` | ✊ | 주먹 |
| 2 | `open_palm` | `OpenPalm` | ✋ | 손바닥 펴기 |
| 3 | `V` | `V` | ✌️ | 브이 (검지+중지) |
| 4 | `pinky` | `Pinky` | 🤙 | 새끼손가락 |
| 5 | `animal` | `Animal` | 🐱 | 애니멀 (손가락으로 귀 만들기) |
| 6 | `k-heart` | `KHeart` | 💗 | K-하트 |

**참고 파일:**
- 모델 클래스 정의: `frontend/public/runtime_sequence/class_names.json`
- 라벨 정규화 로직: `frontend/src/js/gestures.js:167-184`

## 4. 최종 매칭표 (손동작 → 소리)

| 손동작 | 이모지 | 오브젝트 | 사운드 | 재생 방식 |
|--------|--------|----------|--------|-----------|
| **Fist** (주먹) | ✊ | penguin | guitar | 멜로디 시퀀스 |
| **OpenPalm** (손바닥) | ✋ | drum | drum | 원샷 |
| **V** (브이) | ✌️ | tambourine | guitar | 원샷 |
| **Pinky** (새끼손가락) | 🤙 | xylophone | piano | 멜로디 시퀀스 |
| **Animal** (애니멀) | 🐱 | cat | flute | 멜로디 시퀀스 |
| **KHeart** (하트) | 💗 | a | bell | 멜로디 시퀀스 |

### 재생 방식 설명

- **원샷 (oneshot)**: 제스처가 인식될 때 한 번 소리가 재생됩니다.
- **멜로디 시퀀스 (melody)**: 제스처를 유지하는 동안 연속적으로 멜로디가 재생됩니다.

## 5. 인식 설정값

### 5.1 모델 추론 설정

| 항목 | 값 | 설명 |
|------|-----|------|
| `tau` | `0.75` | 제스처 인식 신뢰도 임계값 (tau보다 낮으면 neutral로 전환) |
| `modelIntervalMs` | `60` | 모델 추론 주기 (밀리초) |
| `seq_len` | `8` | 시퀀스 길이 (8프레임 버퍼) |
| `normalization_family` | `pos_scale` | 정규화 방식 |
| `input_type` | `joint63_delta63` | 입력 특성 (관절 좌표 + 델타) |

**참고 파일:**
- `frontend/src/js/model_inference_sequence.js:6-7`
- `frontend/public/runtime_sequence/config.json`

### 5.2 제스처 안정화 설정

| 항목 | 기본값 | Pinky/Animal/KHeart | 설명 |
|------|--------|---------------------|------|
| `CONFIDENCE_ENTER` | `0.58` | `0.34~0.36` | 제스처 진입 신뢰도 |
| `CONFIDENCE_HOLD` | `0.46` | `0.22~0.24` | 제스처 유지 신뢰도 |
| `STABLE_FRAMES` | `1` | `1` | 안정화 프레임 수 |
| `CLEAR_FRAMES` | `1` | `1` | 제스처 해제 프레임 수 |

**참고 파일:**
- `frontend/src/js/gestures.js:80-112`

### 5.3 MediaPipe 설정

| 항목 | 값 | 설명 |
|------|-----|------|
| `numHands` | `1` | 추적할 손 개수 |
| `minHandDetectionConfidence` | `0.50` | 손 검출 최소 신뢰도 |
| `minHandPresenceConfidence` | `0.50` | 손 존재 최소 신뢰도 |
| `minTrackingConfidence` | `0.50` | 손 추적 최소 신뢰도 |
| `inferWidth` | `0` | 추론 해상도 제한 (0=제한 없음) |

**참고 파일:**
- `frontend/src/js/main.js:224` (numHands)
- `frontend/src/js/main.js:987-989, 1010-1012` (confidence 값들)
- `frontend/src/js/hand_tracking_runtime.js:40` (inferWidth)

## 6. 왼손/오른손 처리

### 현재 설정

- **오른손**: 모든 제스처 인식 활성화 ✅
- **왼손**: 제스처 인식 **비활성화** ❌
  - 랜드마크는 추적되지만 모델 추론은 하지 않음
  - 소리도 재생되지 않음

### 왼손 미러링

- `leftHandMirror`: **ON** ✅
- 왼손 랜드마크를 좌우 반전하여 오른손처럼 처리 (현재는 추론 비활성화 상태)

**참고 파일:**
- `frontend/src/js/model_inference_sequence.js:12-16` (leftHandMirror)
- `frontend/src/js/model_inference_sequence.js:101-103` (왼손 추론 비활성화)
- `frontend/src/js/interaction_runtime.js:44-46` (왼손 제스처 비활성화)

## 7. 데이터 흐름

```
[카메라 입력]
    ↓
[MediaPipe HandLandmarker]
    → 21개 손 랜드마크 좌표 추출
    ↓
[hand_tracking_runtime.js]
    → 손 추적 및 캐싱
    ↓
[model_inference_sequence.js]
    → 8프레임 시퀀스 버퍼링
    → pos_scale 정규화 + delta 계산
    → ONNX 모델 추론 (126차원 입력)
    ↓
[gestures.js]
    → 모델 출력 후처리 (tau 필터링, 안정화)
    → 제스처 라벨 결정 (Fist, OpenPalm, V, Pinky, Animal, KHeart)
    ↓
[sound_mapping.js]
    → 제스처 → 오브젝트 매핑
    → 오브젝트 → 사운드 프로필 매핑
    ↓
[interaction_runtime.js]
    → 제스처 반응 처리
    → 쿨다운, 중복 방지
    ↓
[audio.js]
    → 소리 재생 (oneshot 또는 melody sequence)
```
