## 🎯 기술적 차이점

### 1. 입력 데이터 처리

#### 기존 모델
```javascript
// 단순 raw 좌표 전달
입력: [1, 63]  // 21개 관절 × 3차원 (x, y, z)
처리: 즉시 추론 가능
```

#### 새 모델
```typescript
// 정규화 + 시퀀스 + 델타 처리
1단계: pos_scale 정규화
  normalized = (points - points[0]) / ||points[9] - points[0]||

2단계: 8프레임 버퍼 유지
  buffer = [frame_t-7, frame_t-6, ..., frame_t]

3단계: 델타(차분) 계산
  delta[t] = frame[t] - frame[t-1]

4단계: 결합 및 추론
  입력: [1, 8, 126]  // 8프레임 × (관절63 + 델타63)
```

### 2. 인식 가능한 손동작 (동일)

```json
  "V",            // 3: 브이
  "pinky",        // 4: 새끼손가락
  "animal",       // 5: 애니멀
  "k-heart"       // 6: K-하트
]
```

### 3. 상태 관리

#### 기존 모델
```javascript
상태: 없음 (stateless)
처리: 매 프레임 독립적 추론
```

#### 새 모델
```typescript
상태 종류:
```

## ✅ 장점 (새 모델의 이점)

### 1. 시간적 맥락 인식 🎯

**기존**: 단일 프레임만 보고 판단 (현재 손모양만)
**새것**: 최근 8프레임을 보고 판단 (손의 움직임 패턴 파악)

**효과**:
- 동작 인식 정확도 향상 (정적 손모양 + 동적 움직임)
- 순간적인 노이즈에 덜 민감
- 연속 동작의 흐름을 이해

**예시**:
```
손을 천천히 주먹 쥐기 → "fist" 인식 (높은 신뢰도)
카메라 흔들림으로 순간 변형 → 버퍼로 보정 (오인식 방지)
```

### 2. 델타(차분) 특성 추가

**기존**: 관절 위치만 사용 (63차원)
**새것**: 관절 위치 + 프레임 간 변화량 (126차원)

**효과**:
- 움직임 속도/방향 감지 가능
- "빠른 주먹"과 "느린 주먹" 구분 가능
- 제스처 구별력 향상

**예시**:
```
정지한 V → 인식
움직이는 V → 델타로 추가 정보 획득
유사한 손모양도 움직임으로 구분
```

### 3. 위치 불변성 정규화 (pos_scale)

**기존**: raw 좌표 그대로 사용
**새것**: 손목(pt[0]) 기준 정규화 + 손 크기(pt[9]) 스케일 보정

**수식**:
```
normalized = (points - points[0]) / ||points[9] - points[0]||
```

**효과**:
- 카메라 거리 영향 감소 (가까이/멀리 동일)
- 손 위치 변화에 강건 (화면 어디든 안정적)
- 손 크기 차이 보정 (어른/어린이)

### 4. 더 강력한 필터링 (Tau 임계값)

**기존**: tau 설정 없음 (config에서 null)
**새것**: tau=0.85 (85% 미만 자동 중립 처리)

**효과**:
- 오작동 감소 (애매한 동작은 무시)
- 의도적 제스처만 인식
- 사용자 경험 향상

**예시**:
```
손 올리는 중 (전환 동작) → 신뢰도 60% → neutral 처리 ✅
확실한 주먹 → 신뢰도 92% → fist 인식 ✅
```

## ❌ 단점 (새 모델의 트레이드오프)

### 1. 초기 지연 (Warmup) ⏳

**기존**: 첫 프레임부터 바로 추론 가능
**새것**: 8프레임 수집해야 추론 시작

**영향**:
- 손을 올린 후 약 **0.5초 지연** (15 FPS 기준)
- 손 사라진 후 다시 올릴 때마다 warmup 반복
- 빠른 on/off 동작에 불리

**시나리오**:
```
1프레임: warmup (대기)
2프레임: warmup (대기)
...
7프레임: warmup (대기)
8프레임: 첫 추론 시작 ✅
```

### 2. 메모리 사용량 증가 📦

| 항목 | 기존 | 새것 | 증가율 |
|------|------|------|--------|
| 모델 크기 | 134 KB | 1.2 MB | **9배** |
| 버퍼 메모리 | 252 bytes | 2,016 bytes | **8배** |
| 총 메모리 | ~1 MB | ~2.5 MB | **2.5배** |

**영향**:
- 초기 로딩 시간 증가 (모델 다운로드)
- 런타임 메모리 증가 (프레임 버퍼)
- 저사양 기기에서 부담

### 3. 계산 복잡도 증가 💻

**기존**:
```
입력: [1, 63]
연산: 단일 MLP 통과 (~3-5ms)
```

**새것**:
```
입력: [1, 8, 126]
연산: 시퀀스 처리 + 델타 계산 (~5-10ms)
복잡도: 약 2배 증가
```

**영향**:
- 프레임당 추론 시간 증가 (2~5ms 추가)
- CPU/GPU 사용률 증가
- 모바일에서 배터리 소모 증가

### 4. 통합 복잡도 증가 🔧

**기존**:
```javascript
// 간단한 통합
import { getModelPrediction } from "./model_inference.js";
const result = getModelPrediction(landmarks);
```

**새것**:
```typescript
// 복잡한 통합
import { createSequenceDeltaPredictor } from "./inference";

const predictor = createSequenceDeltaPredictor();
await predictor.load();

// 손 있을 때
const result = await predictor.pushFrame(landmarks);

// 손 없을 때
const noHandResult = predictor.pushNoHand();
```

**영향**:
- TypeScript 빌드 설정 필요
- 상태 관리 로직 추가 필요
- `pushNoHand()` 호출 타이밍 관리
- 개발/유지보수 복잡도 증가

### 5. 버퍼 리셋 문제 🔄

**문제 시나리오**:
```
손 올림 → 8프레임 수집 → 추론 시작
손 잠깐 사라짐 → 버퍼 리셋 (pushNoHand)
다시 손 올림 → 다시 8프레임 대기 (warmup)
```

**영향**:
- 빠른 손 움직임 시 인식 끊김 가능
- 카메라 각도 변경 시 손 놓침 → 리셋
- MediaPipe 손 검출 실패 시 버퍼 날아감

## 🎯 사용 권장 시나리오

### ✅ 새 모델을 사용해야 하는 경우

1. **동작 기반 게임/인터랙션**
   - 손의 움직임 패턴이 중요한 경우
   - 예: 악기 연주, 춤 게임, 스포츠 시뮬레이션

2. **정확도가 최우선**
   - 오인식을 최소화해야 하는 경우
   - 예: 의료/교육 앱, 전문가 도구

3. **고사양 환경**
   - 데스크톱 브라우저
   - 최신 모바일 기기 (2020년 이후)

4. **연속 동작 인식**
   - 손을 계속 올려두고 사용하는 앱
   - warmup 한 번만 발생하므로 영향 적음

### ❌ 기존 모델을 유지해야 하는 경우

1. **즉각적인 반응 필요**
   - 0.5초 지연도 허용 안 되는 경우
   - 예: 리듬 게임, 빠른 선택 UI

2. **저사양 환경**
   - 구형 모바일 (2018년 이전)
   - 저전력 기기, IoT 디바이스

3. **빠른 on/off 동작**
   - 손을 자주 올렸다 내렸다 하는 경우
   - 버퍼 리셋 문제로 인식 끊김

4. **단순 손모양 인식**
   - 움직임 패턴이 중요하지 않은 경우
   - 정적인 손 모양만 필요한 경우

```typescript
const predictors = new Map(); // handKey별 predictor

export async function initSequenceModel(handKey) {
  const predictor = createSequenceDeltaPredictor();
  await predictor.load();
  predictors.set(handKey, predictor);

export async function getModelPrediction(landmarks, now, handKey) {
  const predictor = predictors.get(handKey);
  if (!predictor) return null;

  } else {
    return predictor.pushNoHand();
  }
```

```javascript
});
```

### 5단계: 성능 모니터링

```javascript
// 추론 시간 측정
const perfStats = {
  warmupCount: 0,
  inferenceCount: 0,
  avgInferenceMs: 0,
  maxInferenceMs: 0
};

function trackInference(result, elapsed) {
  if (result.status === "warmup") {
    perfStats.warmupCount++;
  } else {
    perfStats.inferenceCount++;
    perfStats.avgInferenceMs =
      (perfStats.avgInferenceMs * (perfStats.inferenceCount - 1) + elapsed)
      / perfStats.inferenceCount;
    perfStats.maxInferenceMs = Math.max(perfStats.maxInferenceMs, elapsed);
  }
}
```
