### 2. 시퀀스 모델 (8프레임)

**파일**: `/frontend/src/js/model_inference_sequence.js`

**데이터 흐름**:

```javascript
MediaPipe landmarks (21개 점, x/y/z)

    ↓

    ↓

    ↓
ONNX 모델 입력
```

**코드**:
```javascript
// line 464-478
const rawFeatures = sanitizeLandmarks(landmarks, handKey);
// rawFeatures = [x0, y0, z0, ..., x20, y20, z20] (raw)

// 1. pos_scale 정규화 적용!
const normalized63 = normalizePosScale(rawFeatures);

// 2. 버퍼에 저장
handState.jointBuffer.push(normalized63);
```

**정규화 함수** (`normalizePosScale()`, line 365-385):
```javascript
function normalizePosScale(frame63) {
  const normalized = new Float32Array(63);

  // 손목(pt[0])을 원점으로
  const originX = frame63[0];
  const originY = frame63[1];
  const originZ = frame63[2];

  // 중지 MCP(pt[9])까지의 거리로 스케일 정규화
  const dx = frame63[27] - originX;  // pt[9].x - pt[0].x
  const dy = frame63[28] - originY;  // pt[9].y - pt[0].y
  const dz = frame63[29] - originZ;  // pt[9].z - pt[0].z
  const denom = Math.hypot(dx, dy, dz);
  const scale = denom <= EPS ? 1 : 1 / denom;

  // 모든 점에 적용: (pts - pts[0]) / ||pts[9] - pts[0]||
  for (let i = 0; i < 63; i += 3) {
    normalized[i]     = (frame63[i]     - originX) * scale;
    normalized[i + 1] = (frame63[i + 1] - originY) * scale;
    normalized[i + 2] = (frame63[i + 2] - originZ) * scale;
  }

  return normalized;
}
```

**정규화 수식**:
```
normalized = (points - points[0]) / ||points[9] - points[0]||
```

**정규화 여부**: ✅ **적용됨**
- 손목을 원점으로 이동
- 손 크기(손목~중지 MCP 거리)로 스케일 정규화
- 카메라 거리, 손 크기 변화에 강건

## 📈 정규화의 효과

### pos_scale 정규화가 없으면 (기존 모델)

**문제점**:
```
카메라 가까이: x=0.3, y=0.4  → 손이 크게 보임
카메라 멀리:   x=0.5, y=0.5  → 손이 작게 보임

→ 같은 제스처인데 좌표가 달라짐!
```

### pos_scale 정규화가 있으면 (시퀀스 모델)

**효과**:
```
카메라 가까이: normalized = (0.3-0, 0.4-0) / 0.15 = (2.0, 2.67)
카메라 멀리:   normalized = (0.5-0, 0.5-0) / 0.25 = (2.0, 2.0)

→ 상대적 위치는 유지됨! (거리 영향 제거)
```

**장점**:
1. **거리 불변**: 카메라 가까이/멀리 상관없이 일정
2. **크기 불변**: 어른/어린이 손 크기 차이 보정
3. **위치 불변**: 화면 어디에 손이 있든 상관없음

## 🤔 왜 기존 모델은 정규화가 없을까?

### 가능한 이유

1. **학습 데이터가 이미 정규화되지 않음**
   - 기존 모델은 raw 좌표로 학습됨
   - 추론 시에도 동일한 형식 필요

2. **MediaPipe 자체 정규화 의존**
   - MediaPipe는 이미 0~1 범위로 normalize
   - 추가 정규화가 필요 없다고 판단

3. **단순성 우선**
   - 빠른 프로토타입을 위해 전처리 최소화

### 문제점

❌ **환경 변화에 취약**:
- 카메라 거리 변화 시 성능 저하
- 손 크기 차이에 민감
- 화면 위치에 따라 정확도 변동

3. **옵션 3**: 두 버전 모두 유지

   - normalized 입력 모델: 높은 정확도 (새로 학습)

### 장기 (표준화)

**모든 모델 정규화 표준화**:

- 환경 변화에 강건한 모델 구축

```javascript
function normalizePosScale(frame63) {
  const normalized = new Float32Array(63);
  const originX = frame63[0];
  const originY = frame63[1];
  const originZ = frame63[2];

  const dx = frame63[27] - originX;
```
