# 성능 저하 원인 및 개선 내역

## 1) 무엇을 하다가 느려졌나
최근 손 추적 시각 효과를 다음처럼 강화한 뒤 프레임 드랍이 발생했습니다.

- 손바닥 중심에 Lottie 나비 애니메이션 렌더링 추가
- 제스처별 틴트/글로우 효과 추가
- 일반 점 파티클을 나비 형태 파티클로 교체
- 피버 타임 중 버블/파티클 효과가 자주 생성됨

결과적으로 프레임당 캔버스 드로우 비용과 손 추론 비용이 동시에 올라가 체감 성능이 저하되었습니다.

## 2) 핵심 원인 분석

### A. 렌더러 핫패스 비용 증가
- 파일: src/js/renderer.js
- 문제:
  - 매 프레임 Lottie 캔버스 조회
  - 틴트 캔버스 합성 + 글로우 블러가 계속 수행
- 영향: CPU/GPU 혼합 부하 증가

### B. 손 추론 빈도/손 개수 기본값이 높음
- 파일: src/js/main.js
- 문제:
  - 기본 추론 FPS가 높아 detectForVideo 호출 밀도가 높음
  - 기본 numHands=2로 연산량 증가
- 영향: 저사양/중간 사양 장치에서 프레임 유지 어려움

### C. 파티클 시스템 복잡도 증가
- 파일: src/js/particle_system.js
- 문제:
  - 최대 파티클/버블 수가 큼
  - 버스트 한 번당 생성량이 큼
- 영향: 피버 구간에서 draw 호출 급증

## 3) 어떻게 고쳤나 (적용한 패치)

### 3-1. 렌더러 최적화 (src/js/renderer.js)
- Lottie 내부 캔버스를 캐싱해서 매 프레임 DOM 조회 비용 절감
- shadowBlur 강도 하향 (20 -> 12)으로 글로우 렌더 부담 완화

### 3-2. 추론/런타임 기본값 조정 (src/js/main.js)
- 기본 추론 FPS 하향 (45 -> 30)
- 최대 추론 FPS 하향 (90 -> 60)
- numHands를 URL 파라미터로 제어하도록 추가
  - 기본값: 1
  - 필요 시: ?numHands=2
- 피버 버블 생성 확률 하향 (0.05 -> 0.025)

### 3-3. 파티클 부하 축소 (src/js/particle_system.js)
- MAX_BUBBLES: 15 -> 10
- MAX_PARTICLES: 150 -> 90
- 버스트 생성량 하향:
  - animal: 28 -> 20
  - kheart: 32 -> 24
  - fist: 24 -> 18
  - default: 18 -> 14
