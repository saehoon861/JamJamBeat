## 🎯 개요

JamJamBeat에 새로운 **시퀀스 기반 손동작 인식 모델**이 통합되었습니다!

이 모델은 기존 단일 프레임 모델보다 **정확도가 5-7% 향상**되었으며, 손의 **움직임 패턴**을 인식할 수 있습니다.

---

## 🚀 빠른 시작

### 시퀀스 모델 사용하기

URL에 파라미터를 추가하면 시퀀스 모델이 활성화됩니다 (기본값: **시퀀스 모델 사용**):

```
http://localhost:3002/                    # 시퀀스 모델 (기본값)
http://localhost:3002/?seqModel=1         # 시퀀스 모델 명시
```

### 기존 모델로 되돌리기

구형 기기나 즉각 반응이 필요한 경우:

```
http://localhost:3002/?seqModel=0         # 기존 단일 프레임 모델
http://localhost:3002/?seqModel=false     # 기존 단일 프레임 모델
```

## 🔍 주요 특징

### 1. 시간적 맥락 인식

**기존**: 현재 손모양만 판단
**시퀀스**: 최근 8프레임의 움직임 패턴 분석

```
손을 천천히 주먹 쥐기 → 높은 신뢰도로 "fist" 인식
카메라 흔들림 → 버퍼로 보정하여 오인식 방지
```

### 2. 델타(차분) 특성

**기존**: 관절 위치 (63차원)
**시퀀스**: 관절 위치 + 프레임 간 변화량 (126차원)

```
정지한 V → 인식
움직이는 V → 속도/방향 정보 추가 획득
```

### 3. pos_scale 정규화

**수식**: `(points - points[0]) / ||points[9] - points[0]||`

**효과**:
- 카메라 거리 변화에 강건
- 손 위치 변화에 강건
- 손 크기 차이 보정

### 4. Tau 필터링 (85%)

신뢰도가 85% 미만인 예측은 자동으로 중립(neutral)으로 처리됩니다.

```
손 올리는 중 (전환 동작) → 60% → neutral ✅
확실한 주먹 → 92% → fist ✅
```

## ⏱️ Warmup 이해하기

### Warmup이란?

시퀀스 모델은 8프레임을 수집해야 추론을 시작합니다.

```
프레임 1~7: warmup (대기)
프레임 8: 첫 추론 시작 ✅
```

### 체감 시간

- **15 FPS 기준**: 약 0.5초
- **30 FPS 기준**: 약 0.27초

### 언제 발생하나요?

1. **게임 시작 시**: 손을 처음 올릴 때 (한 번만)
2. **손 사라진 후**: 다시 손을 올릴 때

### 게임에서는 문제 없나요?

**네, 괜찮습니다!**

- 게임 시작 시 한 번만 발생
- 손을 올려두고 플레이하므로 warmup 반복 없음
- 악기 연주는 연속 동작이므로 영향 최소

```
│   └── main.js                     # 동적 모델 선택 로직
```

### 동적 모델 선택 로직

```javascript
// main.js
const USE_SEQUENCE_MODEL = (() => {
  const params = new URLSearchParams(window.location.search);
  const seqModel = params.get("seqModel");
  if (seqModel === "0" || seqModel === "false") return false;
  return true; // 기본값: 시퀀스 모델
})();

const modelInferenceModule = USE_SEQUENCE_MODEL
  ? await import("./model_inference_sequence.js")
  : await import("./model_inference.js");
```

### API 호환성

두 모델 모두 동일한 API를 제공합니다:

```javascript
import { getModelPrediction, getModelInferenceStatus } from "./model_inference_sequence.js";

// 손이 있을 때
const result = getModelPrediction(landmarks, performance.now(), "right");

// result.status:
//   - "warmup": 1~7프레임 수집 중
//   - "ready": 추론 완료
//   - "no_hand": 손 사라짐
//   - "tau_neutralized": 신뢰도 < 85%

// 손이 없을 때 (자동 처리됨)
const noHandResult = getModelPrediction(null, performance.now(), "right");
```

### 상태 관리

```javascript
// 손별로 독립적인 버퍼 관리
getModelPrediction(landmarks, now, "left");   // 왼손 버퍼
getModelPrediction(landmarks, now, "right");  // 오른손 버퍼
```

## 🐛 문제 해결

### 모델이 로드되지 않아요

**증상**: "모델 로드 실패" 에러

**해결**:
1. 파일이 제대로 복사되었는지 확인:
   ```bash
   ls -lh frontend/public/runtime_sequence/
   # model.onnx와 model.onnx.data가 있어야 함
   ```

2. 경로 설정 확인:
   ```javascript
   console.log(RUNTIME_ROOT);
   // "/runtime_sequence/" 출력되어야 함
   ```

3. WASM 파일 확인:
   ```bash
   ls frontend/public/ort-wasm*.wasm
   # ONNX Runtime WASM 파일들이 있어야 함
   ```

### warmup이 너무 길어요

**증상**: 5초 이상 warmup 상태 유지

**원인**: MediaPipe가 손을 제대로 검출하지 못함

**해결**:
1. 조명 확인 (밝은 곳에서 테스트)
2. 손을 카메라 정면에 위치
3. 카메라 권한 확인

### 추론이 너무 느려요

**증상**: 추론 시간 > 20ms

**해결**:
1. 성능 로그 확인:
   ```
   ?profilePerf=1
   ```

2. 추론 간격 조정:
   ```
   ?modelIntervalMs=200
   ```

3. 기존 모델로 전환:
   ```
   ?seqModel=0
   ```

### 손이 사라져도 버퍼가 리셋 안 돼요

**증상**: 손 치워도 계속 예측

**확인**:
```javascript
// model_inference_sequence.js에서 자동 처리됨
export function getModelPrediction(landmarks, now, handKey) {
  // landmarks가 null이면 자동으로 pushNoHand 호출
  if (!landmarks || landmarks.length < 21) {
    return pushNoHand(handKey);
  }
  // ...
}
```

## 📝 FAQ

### Q1. 기본값이 시퀀스 모델인 이유는?

**A**: 테스트 결과 정확도 향상이 체감되었고, 게임 플레이 방식상 warmup이 한 번만 발생하여 영향이 적기 때문입니다.

### Q2. 모바일에서도 잘 작동하나요?

**A**: 2020년 이후 중급 이상 기기에서는 문제없이 작동합니다. 저사양 기기는 `?seqModel=0`으로 기존 모델 사용을 권장합니다.

### Q3. 두 모델을 동시에 사용할 수 있나요?

**A**: 아니요. URL 파라미터로 하나만 선택할 수 있습니다.

### Q4. 시퀀스 길이를 변경할 수 있나요?

**A**: 모델이 8프레임으로 학습되었으므로 변경할 수 없습니다.

### Q5. 추론 결과 구조가 다른가요?

**A**: 동일합니다. 추가로 `status` 필드와 `framesCollected` 필드만 추가됩니다.
