## 📍 1단계: sanitizeLandmarks() - 기본 변환

**목적**: MediaPipe 랜드마크를 63차원 배열로 변환

### 입력
```javascript
landmarks = [
  { x: 0.5, y: 0.6, z: -0.02 },  // pt[0] 손목
  { x: 0.52, y: 0.58, z: -0.01 }, // pt[1] 엄지 CMC
  { x: 0.54, y: 0.56, z: 0.0 },   // pt[2] 엄지 MCP
  ...
  { x: 0.48, y: 0.62, z: -0.03 }  // pt[20] 새끼 DIP
]
// 총 21개 점
```

### 처리 과정

#### A. 왼손 미러링 (옵션)

```javascript
// URL 파라미터로 제어 (기본값: 활성화)
const LEFT_HAND_MIRROR_ENABLED = true;

function getMirrorPivotX(landmarks) {
  // 손의 중심선 계산
  const wristX = landmarks[0].x;      // 손목
  const indexMcpX = landmarks[5].x;   // 검지 MCP
  const pinkyMcpX = landmarks[17].x;  // 새끼 MCP

  // 평균으로 피벗 계산
  return (wristX + indexMcpX + pinkyMcpX) / 3;
}

// 왼손일 때 x 좌표 반전
if (handKey === "left" && LEFT_HAND_MIRROR_ENABLED) {
  const pivot = getMirrorPivotX(landmarks);
  mirroredX = clamp(pivot * 2 - rawX, 0, 1);
}
```

**왜 미러링?**
```
오른손:  👍 (엄지가 왼쪽)
왼손:    👍 (엄지가 오른쪽)

미러링 후:
왼손 → 오른손처럼 보이도록 변환
→ 모델이 하나의 손 방향만 학습해도 됨
```

#### B. 63차원 배열 변환

```javascript
const features = [];
for (let i = 0; i < 21; i++) {
  const point = landmarks[i];
  features.push(
    point.x,  // x 좌표
    point.y,  // y 좌표
    point.z   // z 좌표 (깊이)
  );
}
// features = [x0, y0, z0, x1, y1, z1, ..., x20, y20, z20]
```

### 출력
```javascript
rawFeatures = [
  0.5, 0.6, -0.02,   // pt[0]
  0.52, 0.58, -0.01, // pt[1]
  ...
  0.48, 0.62, -0.03  // pt[20]
]
// Float32Array(63)
```

## 🎨 2단계: normalizePosScale() - 정규화

**목적**: 손 크기와 위치를 표준화하여 환경 변화에 강건하게 만듦

### 정규화 수식

```
normalized = (points - points[0]) / ||points[9] - points[0]||
```

**설명**:
- `points[0]`: 손목 (원점으로 사용)
- `points[9]`: 중지 MCP (스케일 기준점)
- `||...||`: 유클리드 거리

### 처리 과정

#### A. 원점 설정

```javascript
// 손목을 원점(0, 0, 0)으로
const originX = frame63[0];  // 손목 x
const originY = frame63[1];  // 손목 y
const originZ = frame63[2];  // 손목 z
```

**효과**:
```
변환 전: 손목이 (0.5, 0.6, -0.02) → 절대 위치
변환 후: 손목이 (0, 0, 0)         → 상대 위치
```

#### B. 스케일 계산

```javascript
// 중지 MCP(pt[9])는 인덱스 27, 28, 29
const dx = frame63[27] - originX;
const dy = frame63[28] - originY;
const dz = frame63[29] - originZ;

// 손목 → 중지 MCP 거리
const denom = Math.hypot(dx, dy, dz);
// denom = sqrt(dx² + dy² + dz²)

// 0으로 나누기 방지
const scale = denom <= 1e-8 ? 1 : 1 / denom;
```

**예시**:
```
손이 가까이 있을 때:
  손목 → 중지 거리 = 0.15
  scale = 1 / 0.15 = 6.67

손이 멀리 있을 때:
  손목 → 중지 거리 = 0.10
  scale = 1 / 0.10 = 10.0

→ 멀리 있을수록 더 크게 확대
→ 결과적으로 손 크기 일정하게!
```

#### C. 모든 점에 적용

```javascript
for (let i = 0; i < 63; i += 3) {
  // x 좌표
  normalized[i]     = (frame63[i]     - originX) * scale;
  // y 좌표
  normalized[i + 1] = (frame63[i + 1] - originY) * scale;
  // z 좌표
  normalized[i + 2] = (frame63[i + 2] - originZ) * scale;
}
```

### 출력 (정규화된 63차원)

```javascript
normalized63 = [
  0, 0, 0,           // pt[0] 손목 (항상 원점)
  0.13, -0.11, 0.05, // pt[1] 상대 위치
  ...
  -0.12, 0.15, -0.06 // pt[20] 상대 위치
]
// Float32Array(63)
```

### 정규화 효과 비교

**시나리오: 같은 주먹 제스처를 가까이/멀리서 촬영**

| 구분 | Raw 좌표 | 정규화 후 |
|------|----------|----------|
| **가까이** | 손목: (0.3, 0.4)<br>검지: (0.35, 0.38) | 손목: (0, 0)<br>검지: (0.33, -0.13) |
| **멀리** | 손목: (0.5, 0.5)<br>검지: (0.53, 0.49) | 손목: (0, 0)<br>검지: (0.30, -0.10) |
| **결과** | ❌ 좌표가 다름 | ✅ 거의 동일! |

## 🗂️ 3단계: 8프레임 버퍼링

**목적**: 시간적 맥락 파악 (손의 움직임 패턴 인식)

### 버퍼 관리

```javascript
// 손별로 독립적인 버퍼
const handState = {
  jointBuffer: [],  // 최대 8개의 정규화된 프레임
  lastRequestAt: 0,
  inFlight: false
};

// 새 프레임 추가
handState.jointBuffer.push(normalized63);

// 8개 초과 시 오래된 프레임 제거 (FIFO)
if (handState.jointBuffer.length > 8) {
  handState.jointBuffer.shift();
}
```

### Warmup 단계

```javascript
if (handState.jointBuffer.length < 8) {
  // 아직 8프레임이 안 차면 "warmup" 상태
  return {
    status: "warmup",
    predLabel: "neutral",
    framesCollected: handState.jointBuffer.length
  };
}
```

**타임라인**:
```
프레임 1: [frame1]                      → warmup (1/8)
프레임 2: [frame1, frame2]               → warmup (2/8)
프레임 3: [frame1, frame2, frame3]       → warmup (3/8)
...
프레임 7: [frame1, ..., frame7]          → warmup (7/8)
프레임 8: [frame1, ..., frame8]          → ready! 첫 추론 ✅
프레임 9: [frame2, ..., frame9]          → ready (최신 8개)
프레임 10: [frame3, ..., frame10]        → ready
```

### 버퍼 리셋

**손이 사라졌을 때**:
```javascript
export function pushNoHand(handKey) {
  const handState = getHandState(handKey);
  handState.jointBuffer = [];  // 버퍼 비움
  return { status: "no_hand", predLabel: "neutral" };
}
```

**시나리오**:
```
1. 손 올림 → 8프레임 수집 → 추론 시작
2. 손 치움 → 버퍼 리셋
3. 다시 손 올림 → 다시 8프레임 수집 필요
```

## 🔧 4단계: buildFeatureTensor() - 델타 특성 생성

**목적**: 정적 위치 + 동적 변화량 결합

### 입력
```javascript
buffer = [
  frame_t-7,  // 8프레임 전
  frame_t-6,
  ...
  frame_t     // 현재 프레임
]
// 각 프레임은 Float32Array(63)
```

### 델타(차분) 계산

```javascript
const featureTensor = new Float32Array(8 * 126);
// 8프레임 × 126차원 = 1,008개 값

for (let t = 0; t < 8; t++) {
  const baseOffset = t * 126;
  const current = buffer[t];
  const previous = t > 0 ? buffer[t - 1] : null;

  for (let i = 0; i < 63; i++) {
    // [0~62]: 관절 위치 (joint)
    featureTensor[baseOffset + i] = current[i];

    // [63~125]: 델타 (delta)
    if (previous === null) {
      // 첫 프레임은 델타가 0
      featureTensor[baseOffset + 63 + i] = 0;
    } else {
      // 프레임 간 변화량
      featureTensor[baseOffset + 63 + i] = current[i] - previous[i];
    }
  }
}
```

### 델타 특성의 의미

**예시: 검지 끝의 y 좌표**

| 프레임 | 위치 (joint) | 델타 (delta) | 해석 |
|--------|-------------|-------------|------|
| t=0 | 0.5 | 0.0 | 첫 프레임 |
| t=1 | 0.48 | -0.02 | ⬇️ 아래로 |
| t=2 | 0.46 | -0.02 | ⬇️ 계속 하강 |
| t=3 | 0.45 | -0.01 | ⬇️ 느려짐 |
| t=4 | 0.45 | 0.0 | ⏸️ 정지 |
| t=5 | 0.47 | +0.02 | ⬆️ 위로 |

**델타의 효과**:
```
정지한 주먹: joint만 있음 → 위치 정보
움직이는 주먹: joint + delta → 위치 + 속도/방향

→ 같은 모양이어도 움직임으로 구분 가능!
```

### 출력 (최종 특성 텐서)

```javascript
featureTensor = Float32Array(1008) = [
  // 프레임 0 (126차원)
  j0, j1, ..., j62,    // 관절 위치 (63개)
  d0, d1, ..., d62,    // 델타 (63개, 모두 0)

  // 프레임 1 (126차원)
  j0, j1, ..., j62,    // 관절 위치
  d0, d1, ..., d62,    // 델타 (frame1 - frame0)

  ...

  // 프레임 7 (126차원)
  j0, j1, ..., j62,    // 관절 위치
  d0, d1, ..., d62     // 델타 (frame7 - frame6)
]
```

**형태 변환**:
```javascript
// [1008] → [1, 8, 126] 로 reshape
const inputTensor = new ort.Tensor(
  "float32",
  featureTensor,
  [1, 8, 126]
);
// batch=1, sequence=8, features=126
```

## 🎯 ONNX 모델 추론

### 입력

```javascript
inputTensor = {
  shape: [1, 8, 126],
  type: "float32",
  data: featureTensor
}
```

**해석**:
- `1`: 배치 크기 (한 번에 한 손)
- `8`: 시퀀스 길이 (8프레임)
- `126`: 특성 차원 (관절 63 + 델타 63)

### 추론 실행

```javascript
const inputName = onnxSession.inputNames[0];  // "input"
const outputName = onnxSession.outputNames[0]; // "output"

const results = await onnxSession.run({
  [inputName]: inputTensor
});

const logits = Array.from(results[outputName].data);
// logits = [0.1, 2.3, -0.5, 1.2, 0.8, -0.3, 0.4]
// 7개 클래스에 대한 로짓 값
```

### 출력 처리

```javascript
// Softmax로 확률 변환
const probs = softmax(logits);
// probs = [0.08, 0.72, 0.04, 0.24, 0.16, 0.05, 0.10]

// 최대 확률 클래스 선택
const predIndex = probs.indexOf(Math.max(...probs));
// predIndex = 1 (fist)

const confidence = probs[predIndex];
// confidence = 0.72 (72%)

// Tau 필터링 (85% 미만은 neutral로)
if (confidence < 0.85) {
  predIndex = 0;  // neutral
  tauNeutralized = true;
}
```

```javascript

  predIndex: 1,
  confidence: 0.72,
  probs: [0.08, 0.72, 0.04, 0.24, 0.16, 0.05, 0.10],
  tauNeutralized: true,  // 85% 미만

  elapsed_ms: 6.2
}
```

## 💡 핵심 포인트

### 1. 정규화가 중요한 이유

**정규화 없으면**:
```
가까이 촬영: (0.3, 0.4) → 모델이 "큰 손"으로 인식
멀리 촬영:   (0.5, 0.5) → 모델이 "작은 손"으로 인식
→ 같은 제스처인데 다르게 인식! ❌
```

**정규화 있으면**:
```
가까이 촬영: (0, 0) 기준 상대 위치
멀리 촬영:   (0, 0) 기준 상대 위치
→ 거리 무관하게 일정! ✅
```

### 2. 델타 특성의 중요성

**관절 위치만**:
```
주먹 (정지) vs 주먹 (펀치)
→ 순간 위치는 비슷 → 구분 어려움
```

**관절 위치 + 델타**:
```
주먹 (정지): delta ≈ 0
주먹 (펀치): delta >> 0 (빠른 전진)
→ 움직임으로 구분 가능! ✅
```

### 3. 8프레임 버퍼의 의미

```
1프레임: 스냅샷 (정적)
8프레임: 짧은 동영상 (동적)
→ 움직임 패턴, 흐름 파악 가능
```

**예시**:
- V 모양 만들기: 손가락 펴지는 과정 인식
- 주먹 쥐기: 손가락 접히는 과정 인식
