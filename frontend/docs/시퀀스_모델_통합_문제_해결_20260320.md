## 🐛 발생한 문제

### 문제 1: Top-level await 에러

**증상**:
```
Transform failed with 2 errors:
assets/main.a671798c.js:4991:4: ERROR: Top-level await is not available in the configured target environment ("es2020")
```

**원인**:
- ES2020 타겟에서는 top-level await가 지원되지 않음
- `main.js`에서 모듈 레벨에서 직접 `await import()` 사용

**문제 코드**:
```javascript
// main.js (잘못된 코드)
const modelInferenceModule = USE_SEQUENCE_MODEL
  ? await import("./model_inference_sequence.js")
  : await import("./model_inference.js");

const { getModelPrediction, getModelInferenceStatus } = modelInferenceModule;
```

**해결 방법**:
`init()` 함수 내부로 동적 import를 이동하여 함수 내 await 사용

```javascript
// main.js (수정된 코드)
// 모듈 레벨에서는 변수만 선언
let getModelPrediction = null;
let getModelInferenceStatus = null;

async function init() {
  // init 함수 내부에서 동적 로딩
  const modelInferenceModule = USE_SEQUENCE_MODEL
    ? await import("./model_inference_sequence.js")
    : await import("./model_inference.js");

  getModelPrediction = modelInferenceModule.getModelPrediction;
  getModelInferenceStatus = modelInferenceModule.getModelInferenceStatus;

  // 나머지 초기화 로직...
}
```

**교훈**:
- Top-level await는 ES2022+에서만 지원
- ES2020 타겟에서는 async 함수 내부에서만 await 사용 가능

### 문제 2: getModelPrediction is not a function

**증상**:
```
HandTracking loop error: TypeError: getModelPrediction is not a function
    at Object.processGestureTriggers (interaction_runtime.js:359:22)
```

**원인**:
- `interactionRuntime` 생성 시점에 `getModelPrediction`이 아직 `null`
- `init()` 함수에서 모델을 동적 로딩하기 전에 런타임 객체들이 먼저 생성됨

**문제 코드**:
```javascript
// main.js (잘못된 코드)
let getModelPrediction = null; // 초기값 null

const interactionRuntime = createInteractionRuntime({
  // ...
  getModelPrediction, // null이 전달됨!
  // ...
});

async function init() {
  // 여기서 비로소 할당됨
  getModelPrediction = modelInferenceModule.getModelPrediction;
}
```

**해결 방법**:
함수 참조 방식으로 변경하여 런타임에 동적으로 해석되도록 수정

```javascript
// main.js (수정된 코드)
const interactionRuntime = createInteractionRuntime({
  // ...
  // 화살표 함수로 감싸서 런타임에 평가되도록 변경
  getModelPrediction: (...args) => getModelPrediction?.(...args),
  // ...
});

async function init() {
  // 나중에 할당되어도 정상 동작
  getModelPrediction = modelInferenceModule.getModelPrediction;
}
```

**핵심 차이**:
```javascript
// ❌ 나쁜 예: 생성 시점의 null 값이 고정됨
getModelPrediction: getModelPrediction

// ✅ 좋은 예: 호출 시점에 현재 값을 참조
getModelPrediction: (...args) => getModelPrediction?.(...args)
```

**교훈**:
- JavaScript는 값을 복사하지 참조를 유지하지 않음
- 나중에 할당될 변수는 화살표 함수로 감싸서 전달
- Optional chaining (`?.`)으로 null 체크도 함께 처리

### 문제 3: admin.html 빌드 실패

**증상**:
```
Could not resolve entry module (admin.html).
Error: Could not resolve entry module (admin.html).
```

**원인**:
- `vite.config.mjs`에 `admin.html`이 입력 파일로 설정되어 있음
- 실제로는 `admin.html` 파일이 존재하지 않음

**문제 코드**:
```javascript
// vite.config.mjs (잘못된 코드)
build: {
  rollupOptions: {
    input: {
      index: resolve(rootDir, "index.html"),
      theme: resolve(rootDir, "theme.html"),
      admin: resolve(rootDir, "admin.html") // 파일이 없음!
    }
  }
}
```

**해결 방법**:
존재하지 않는 파일을 빌드 설정에서 제거

```javascript
// vite.config.mjs (수정된 코드)
build: {
  rollupOptions: {
    input: {
      index: resolve(rootDir, "index.html"),
      theme: resolve(rootDir, "theme.html")
      // admin.html 제거
    }
  }
}
```

**교훈**:
- 빌드 설정과 실제 파일 구조를 일치시킬 것
- 사용하지 않는 엔트리 포인트는 제거

## 📚 배운 점

### 1. JavaScript 동적 import 패턴

**안티패턴**:
```javascript
// top-level await (ES2022+만 지원)
const module = await import("./module.js");
```

**권장 패턴**:
```javascript
// async 함수 내부에서 await
async function init() {
  const module = await import("./module.js");
}
```

### 2. 지연 초기화 변수 전달

**안티패턴**:
```javascript
let value = null;
const obj = { fn: value }; // null 고정
value = actualFunction;     // obj.fn은 여전히 null
```

**권장 패턴**:
```javascript
let value = null;
const obj = { fn: (...args) => value?.(...args) }; // 동적 참조
value = actualFunction;     // obj.fn() 호출 시 actualFunction 실행
```

### 2. 모듈 로딩 상태 관리

```javascript
const modelState = {
  loading: false,
  loaded: false,
  error: null
};

async function loadModel() {
  if (modelState.loading) return;
  modelState.loading = true;

  try {
    const module = await import("./model.js");
    modelState.loaded = true;
    return module;
  } catch (error) {
    modelState.error = error;
  } finally {
    modelState.loading = false;
  }
}
```

### 3. 설정 파일 자동 검증

```javascript
// vite.config.mjs
import { existsSync } from 'fs';

function validateInputs(inputs) {
  Object.entries(inputs).forEach(([key, path]) => {
    if (!existsSync(path)) {
      throw new Error(`Entry file not found: ${key} -> ${path}`);
    }
  });
}

const inputs = {
  index: resolve(rootDir, "index.html"),
  theme: resolve(rootDir, "theme.html")
};

validateInputs(inputs);

export default {
  build: {
    rollupOptions: { input: inputs }
  }
};
```
