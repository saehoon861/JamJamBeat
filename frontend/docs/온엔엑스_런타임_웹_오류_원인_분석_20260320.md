# ONNX Runtime Web 오류 원인 정리 (2026-03-20)

## 한 줄 요약
이전 장애의 직접 원인은 외부 데이터 분리 ONNX 모델(model.onnx + model.onnx.data) 로딩 시 externalData 매핑 누락이었고, 여기에 배포 경로(base path)와 WASM 경로 불일치가 겹치면서 초기화 실패가 반복된 것입니다.

## 당시 관찰된 대표 에러
~~~text
[ModelInferenceONNX] 모델 로드 실패: Error: Cannot create a session.
ERROR_CODE: 1, ERROR_MESSAGE: Deserialize tensor embed.1.bias failed.
Failed to load external data file model.onnx.data,
error: Module.MountedFiles is not available.
~~~

## 실제 원인 체인
1. model.onnx가 가중치를 model.onnx.data로 분리 참조하는 구조였음.
2. 브라우저에서 InferenceSession.create(MODEL_PATH)만 호출하면 외부 데이터 파일 해석이 환경에 따라 실패할 수 있음.
3. ONNX 세션 생성이 실패하면 기본 gestureMode=model 흐름에서 제스처 인식 불가가 즉시 노출됨.
4. 추가로 배포 base path와 /runtime/*, WASM 경로가 맞지 않으면 404나 WASM 부팅 실패가 함께 발생해 오류가 확대됨.

## 왜 백엔드 없이 브라우저 추론에서 특히 잘 터졌나
- 브라우저 추론은 정적 파일 서빙 URL과 경로 해석에 전적으로 의존함.
- 모델 본체(model.onnx) 접근이 가능해도 외부 데이터(model.onnx.data) 또는 WASM 파일 경로가 어긋나면 세션 생성이 실패함.

## 적용한 수정
### 1) 외부 데이터 명시 매핑
src/js/model_inference.js의 ONNX 세션 생성 옵션에 externalData를 명시함.

~~~js
onnxSession = await ort.InferenceSession.create(MODEL_PATH, {
  executionProviders: ["wasm"],
  graphOptimizationLevel: "all",
  externalData: [
    {
      path: "model.onnx.data",
      data: "/runtime/model.onnx.data"
    }
  ]
});
~~~

### 2) 런타임과 wasm 경로 해석 보강
- runtimeRoot: query, global, env, base-url 우선순위로 동적 해석
- ortWasmRoot: 로컬 경로 우선 사용, 필요 시 CDN fallback

### 3) 운영 디버깅 로그 강화
- 모델 로드 성공 시 runtimeRoot, wasmBase를 함께 기록

## 재발 방지 체크리스트
- 배포가 루트 경로가 아니면 runtimeRoot, ortWasmRoot를 명시한다.
- 배포 후 Network 탭에서 아래 요청이 모두 200인지 확인한다.
  - .../runtime/model.onnx
  - .../runtime/model.onnx.data
  - .../ort-wasm-*.wasm
  - .../mediapipe/vision_wasm_internal.wasm
- 콘솔에서 ModelInferenceONNX 성공 로그와 mode=onnx-local 상태를 확인한다.
