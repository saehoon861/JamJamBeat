# 왼손 추론 방식 정리

## 개요

현재 제스처 모델은 **오른손 데이터 중심으로 학습된 모델**입니다.
그래서 왼손 랜드마크를 그대로 넣으면, 모델 입장에서는 학습 때 보지 못한 형태가 들어오게 됩니다.

이 문제를 줄이기 위해 프론트엔드에서는 **왼손 랜드마크를 오른손 형태로 미러링한 뒤 같은 모델에 넣는 방식**을 사용합니다.

즉, 현재 구조는 아래와 같습니다.

```text
오른손 -> 원본 랜드마크 그대로 추론
왼손   -> X축 반사(미러링) 후 추론
```

---

## 왜 이렇게 하는가

오른손만 학습된 모델은 손가락의 상대 배치가 오른손 기준으로 익혀져 있습니다.

예를 들어:

- 오른손 브이 모양
- 왼손 브이 모양

사람이 보기에는 같은 동작이지만, 랜드마크 좌표만 보면 좌우가 뒤집힌 다른 패턴입니다.

그래서 왼손을 그대로 넣으면:

- 다른 클래스로 잘못 분류되거나
- 신뢰도가 낮아져 `neutral` 처리되거나
- 동작 유지가 불안정해질 수 있습니다

이를 보완하려고 **왼손 입력을 오른손처럼 보이게 변환**해서 추론합니다.

---

## 현재 추론 흐름

### 1. MediaPipe가 손을 감지

런타임에서 손마다 `left` 또는 `right` 라벨을 붙입니다.

관련 파일:

- `frontend/src/js/hand_tracking_runtime.js`

핵심 역할:

- 손 랜드마크 추출
- 손별 `handKey` 부여
- 각 손을 별도 상태로 추적

---

### 2. 모델 추론 전에 손별 전처리 수행

모델 입력 직전에 `sanitizeLandmarks(landmarks, handKey)`가 호출됩니다.

관련 파일:

- `frontend/src/js/model_inference.js`
- `frontend/src/js/model_inference_onnx.js`

이 함수는 21개 랜드마크를 `63차원(x, y, z)` 배열로 바꾸기 전에,
`handKey === "left"` 인 경우 X축을 반사합니다.

---

## 왼손 미러링 방식

### 기본 아이디어

왼손의 X 좌표를 손 중심 기준으로 뒤집어,
오른손과 비슷한 공간 배치가 되도록 만듭니다.

현재 구현은 손 중심 X값을 아래 3개 포인트의 평균으로 잡습니다.

- wrist: `landmarks[0]`
- index MCP: `landmarks[5]`
- pinky MCP: `landmarks[17]`

공식은 다음과 같습니다.

```text
mirrorPivotX = (wrist.x + indexMcp.x + pinkyMcp.x) / 3
mirroredX = mirrorPivotX * 2 - rawX
```

그리고 화면 좌표 범위를 벗어나지 않도록 `0~1`로 clamp 합니다.

```text
mirroredX = clamp(mirrorPivotX * 2 - rawX, 0, 1)
```

Y, Z 값은 그대로 사용합니다.

즉:

- `x`: 왼손일 때만 반사
- `y`: 그대로 사용
- `z`: 그대로 사용

---

## 실제 입력 예시

### 오른손

```text
[x1, y1, z1, x2, y2, z2, ... x21, y21, z21]
```

오른손은 원본 좌표 그대로 모델에 들어갑니다.

### 왼손

```text
[mirror(x1), y1, z1, mirror(x2), y2, z2, ... mirror(x21), y21, z21]
```

왼손은 X값만 반사한 뒤 모델에 들어갑니다.

---

## 장점

- 오른손만 학습한 모델을 바로 활용할 수 있습니다
- 왼손 인식 품질이 그대로 넣는 것보다 일반적으로 더 좋아집니다
- 재학습 없이 빠르게 적용 가능합니다
- 런타임 수정만으로 실험할 수 있습니다

---

## 한계

이 방법은 **완전한 해결책은 아닙니다**.

이유:

- 왼손과 오른손은 실제 사용 습관이 다를 수 있음
- 손목 회전, 카메라 각도, 깊이(Z) 패턴은 완전히 대칭이 아닐 수 있음
- 일부 제스처는 단순 X축 반사만으로는 충분하지 않을 수 있음

그래서 현재 방식은:

- **실전용 임시 보정**
- **재학습 전 성능 개선용 우회 방법**

으로 보는 것이 맞습니다.

---

## 가장 좋은 최종 방법

정확도를 가장 안정적으로 높이려면 아래 순서가 좋습니다.

### 1. 왼손 데이터 추가 수집

- 같은 클래스
- 같은 촬영 조건
- 같은 전처리 규칙

으로 왼손 데이터셋을 추가합니다.

### 2. 학습 시 좌우 반전 augmentation 추가

오른손 샘플을 좌우 반전해 학습에 섞으면,
모델이 좌우 대칭 패턴을 더 잘 일반화할 수 있습니다.

### 3. 필요하면 handedness를 모델 입력에 포함

현재 모델은 손 방향 자체를 별도 feature로 받지 않습니다.
향후에는 `left/right` 정보를 feature로 추가하는 방식도 고려할 수 있습니다.

## 현재 토글 방법

왼손 미러링은 기본적으로 켜져 있습니다.

### 기본값

```text
leftHandMirror = on
```

### 끄는 방법

URL 파라미터로 아래처럼 끌 수 있습니다.

```text
http://localhost:3002/?leftHandMirror=0
```

또는

```text
http://localhost:3003/?leftHandMirror=false
```

이렇게 하면 왼손도 원본 좌표 그대로 모델에 들어갑니다.

## 요약

현재 왼손 추론은 아래 방식입니다.

1. MediaPipe가 손을 `left/right`로 구분합니다.
2. 오른손은 원본 랜드마크 그대로 모델에 넣습니다.
3. 왼손은 X축만 손 중심 기준으로 반사합니다.
4. 반사된 왼손 랜드마크를 오른손 학습 모델에 넣습니다.
5. 그 결과를 왼손 제스처 결과로 사용합니다.

즉, 현재 왼손 추론은 **왼손을 오른손 형태로 보정해서 같은 모델로 추론하는 구조**입니다.
