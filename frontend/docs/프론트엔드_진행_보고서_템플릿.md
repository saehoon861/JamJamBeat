# JamJamBeat 프론트엔드 진행 보고서 템플릿

## 문서 목적

이 문서는 `교수/발표용` 기준으로 프론트엔드 작업을 날짜 구간 단위로 정리할 때 사용하는 템플릿이다.
각 기간마다 `작업 목표`, `수행 내용`, `산출물/결과`, `이슈 및 해결`, `비고`를 같은 형식으로 기록해
언제부터 언제까지 무엇을 했는지 한눈에 보이도록 구성한다.

기본 기록 축은 아래 범주를 기준으로 잡는다.

- UI/화면 구성
- 손 추적 및 제스처 인식 연동
- 모델 추론 연결
- 사운드/애니메이션/인터랙션
- 성능 최적화 및 오류 수정
- 테스트 및 배포 준비

---

## 작성 규칙

- 날짜는 반드시 `YYYY-MM-DD ~ YYYY-MM-DD` 형식으로 작성한다.
- 기간별 블록은 같은 6개 항목을 유지한다.
- `수행 내용`은 동사형으로 작성한다.
- `산출물/결과`는 완료 상태가 드러나도록 작성한다.
- `이슈 및 해결`은 문제와 대응을 함께 적는다.
- 발표용 문서이므로 설명은 짧고 명확하게 유지한다.

- 사용자가 웹캠으로 손 동작을 인식하고 바로 반응을 확인할 수 있는 메인 화면 구현

- 모델 추론 결과를 사운드, 오브젝트, 애니메이션과 연동
- 실시간 처리 성능을 개선하고 주요 오류 상황에 대응

- 작업 목표:

  - 웹캠 입력 흐름과 사용자 시작 동선 정리

- 수행 내용:
  - 메인 페이지 레이아웃과 시작 화면 구조를 작성
  - 버튼, 안내 문구, 배경 비디오, 기본 스타일을 적용

  - 시연용 기본 화면 골격을 확보

- 이슈 및 해결:
  - 브라우저별 카메라 접근 실패 상황이 발생
  - 권한 안내 문구와 예외 처리 분기를 추가해 초기 진입 실패를 줄임

  - 프레임 단위 랜드마크 캐시와 좌표 처리 흐름을 구성

  - 실시간 손 랜드마크 추출이 가능해짐
  - 제스처 추론을 위한 입력 데이터 흐름을 확보

- 모바일 및 다양한 해상도 대응 강화
- 로딩 속도와 렌더링 성능 추가 최적화
- 사용자 피드백을 반영한 UI/UX 보완

## 검증 체크리스트

아래 항목을 만족하면 템플릿이 올바르게 채워진 것으로 본다.

- 기간이 빠짐없이 순서대로 정리되어 있다.
- 각 기간마다 수행 내용이 2개 이상 적혀 있다.
- 각 기간마다 산출물 또는 결과가 명시되어 있다.
- 이슈와 해결이 함께 정리되어 있다.
- 마지막에 전체 작업 요약표가 있다.
- 마지막에 최종 성과와 향후 보완 사항이 있다.

## 기본 가정

- 문서 대상은 `교수/발표용`으로 가정한다.
- 작업 기록 단위는 `날짜 구간 단위`로 고정한다.
- 문서 목적은 상세 구현 설계보다 진행 내역 정리와 보고에 둔다.
- 프론트엔드 핵심 범주는 `UI`, `손 추적`, `추론 연동`, `사운드/애니메이션`, `최적화` 중심으로 잡는다.
