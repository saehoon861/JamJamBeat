### Analysis of Initial Results

초기 9개 조합 실험 결과, 일부 조합에서는 validation accuracy가 비교적 높게 나타났지만,  
**test accuracy와의 일관성이 충분하지 않았고**, 조합에 따라 성능 편차도 크게 나타났다.

특히 다음과 같은 경향을 확인할 수 있었다.

- `BatchNorm=False` 조건에서는 하이퍼파라미터 변화에 따라 성능 변동 폭이 컸다.
- validation 성능이 좋아 보여도 test 성능이 안정적으로 따라오지 않는 경우가 있었다.
- 특정 조합은 validation accuracy가 높더라도 실제 일반화 성능(test accuracy)은 상대적으로 낮게 나타났다.

이러한 결과를 바탕으로, 초기 9개 조합 중 **상대적으로 성능이 좋았던 Top 2 조합**을 선정하였다.

## Additional Experiment with Random Rate

선정된 Top 2 조합을 대상으로,  
데이터 분할 또는 샘플링 과정의 랜덤성에 따른 영향을 확인하기 위해 **random rate를 변경하며 추가 실험**을 수행하였다.

그러나 이 단계에서는 다음과 같은 문제가 확인되었다.

- random rate가 달라질 때마다 성능이 일정하지 않았다.
- 동일한 하이퍼파라미터 조합이라도 성능 변동이 발생하였다.
- 즉, 모델이 데이터 분할 및 샘플링의 랜덤성에 민감하게 반응하는 경향을 보였다.

이는 초기 설정이 **충분히 안정적인 학습 구조를 제공하지 못하고 있을 가능성**을 시사한다.

---

## Effect of Batch Normalization

이후 모델에 **Batch Normalization**을 추가하여 다시 실험한 결과,  
성능이 **약 0.79대까지 상승**하는 것을 확인하였다.

또한, Batch Normalization 적용 후에는 단순한 성능 향상뿐 아니라 다음과 같은 점이 함께 확인되었다.

- random rate를 변경하더라도 성능이 이전보다 훨씬 안정적으로 유지되었다.
- 실험 간 편차가 줄어들어 결과 재현성과 신뢰성이 높아졌다.
- 즉, Batch Normalization이 **성능 향상**과 **학습 안정성 확보**에 모두 긍정적인 영향을 준 것으로 해석할 수 있다.

---

## Conclusion

초기 9개 조합 실험에서는 일부 조합이 비교적 좋은 성능을 보였으나,  
random rate 변경에 따라 성능이 일정하지 않아 **모델의 안정성이 충분하지 않음**을 확인하였다.

반면, **Batch Normalization을 추가한 이후에는 test accuracy가 0.79대까지 향상되었고**,  
random rate를 변경하더라도 성능이 일정 수준 유지되는 경향이 나타났다.

따라서 본 실험에서는 Batch Normalization이 단순한 성능 개선 요소일 뿐 아니라,  
**학습 결과의 안정성과 일반화 성능을 높이는 핵심 요소**로 작용한 것으로 판단된다.
